# 프로젝트 요약

## 개요

사업자등록증 정보 추출기는 사업자등록증 이미지, PDF, ZIP 파일을 업로드해 거래처 등록에 필요한 정보를 자동 추출하는 로컬 웹 애플리케이션입니다.

## 주요 기능

- 여러 파일 및 ZIP 파일 일괄 업로드
- PDF 텍스트 추출 및 이미지 렌더링 기반 AI OCR
- Ubion LiteLLM 연동 비전 모델 사용
- JSON 후처리와 품질 검사를 통한 결과 정규화
- 카카오 주소 검색 API를 이용한 우편번호 보정
- 사내 거래처 등록 양식에 맞춘 엑셀 다운로드
- 화면 푸터에 현재 사용 중인 AI 모델명 표시

## 현재 기본 모델

```env
UBION_VISION_MODEL=gpt-4o
```

## 핵심 파일

- `server.ts`: Express 서버, 파일 업로드, AI OCR, 카카오 우편번호 보정
- `src/App.tsx`: React 화면, 파일 큐, 결과 그리드, 엑셀 다운로드
- `.env.example`: 로컬 실행에 필요한 환경 변수 예시
- `run_local.bat`: Windows 로컬 실행 스크립트

## 보안 주의

`.env`에는 실제 API 키가 들어가므로 GitHub에 올리지 않습니다. 공개 저장소에는 `.env.example`만 포함합니다.

# 📋 사업자등록증 정보 추출기 - 개발 및 작업 완료 보고서

이 문서는 **'사업자등록증 정보 추출기 (Business Registration Certificate Information Extractor)'** 애플리케이션의 개발 작업 내역과 특징, 그리고 기술 사양을 정리한 기술 산출물입니다.

---

## 1. 🌟 서비스 개요
본 애플리케이션은 사용자가 업로드한 사업자등록증 이미지(PNG, JPG 등) 또는 PDF 파일에서 핵심 정보를 빠르고 정확하게 분석·추출하여 데이터베이스에 바로 적재하거나 엑셀 파일로 바로 다운로드할 수 있도록 돕는 스마트 고성능 추출기입니다.

- **대상 플랫폼:** Web Application (React, Express, Vite, Node.js)
- **주요 기능:** 사업자등록증 문서 분석 OCR, 상세 주소 정제 및 카카오 우편번호 조회 연동, 일괄 파일 처리 및 다운로드

---

## 2. 🛠️ 핵심 기능 및 개선 내용

### ① AI 기반 고성능 OCR 정보 추출 (`server.ts`)
- **기술 스택:** `@google/genai` 인터페이스 및 `gemini-3.5-flash` 모델 도입
- **정확한 데이터 정형화:** 인공지능이 무작위 텍스트가 아닌 사전에 합의된 데이터 형식(Schema)에 맞추어 `JSON` 형태로 오차 없이 정보를 변환합니다.
- **주요 필드 자동 채움 & 변환 조건:**
  - **회사코드 (`Company_Code`):** 비즈니스 규칙에 따라 항상 `"1000"`으로 고정 추출
  - **거래처 구분 (`Client_Type`):** 항상 `"1"`로 고정 추출
  - **거래처 코드 (`Client_Code`):** 공백 문자열(`""`)로 설정
  - **거래처 약칭 (`Company_Name_Abbrev`):** 법인 종류(예: (주)) 등 탈락 없이 마스터 상호명 명칭(`Company_Name`)과 100% 동일하게 일치
  - **업태 (`Business_Type`) 및 종목 (`Business_Sector`):** 쉼표나 개행 등으로 여러 개가 기재되어 있는 경우 중복 수집을 필터링하고 **가장 상위에 있는(가장 먼저 나열된) 대표 단어 한 가지만** 깔끔하게 추출하도록 고도화
  - **주소지 (`Address_Detail_1`):** 사업자등록증상 기재된 사업장 주소 전체를 완벽 분석

### ② 카카오 주소 검색 API (`KAKAO_REST_API_KEY`) 연동 고도화
- **정교한 우편번호 추출:** 단순히 텍스트에서 5자리 숫자를 무작위 검색하는 대신, 추출된 도로명/지번 주소를 지능적으로 정제(동/호수, 괄호 안 건물명 등 노이즈 문자열 제거)하여 **카카오 로컬 API(Kakao Local API/Address)**에 직접 쿼리합니다.
- **검색 정밀도 상승:** 이 특수 전처리 로직 덕분에 주소지가 길거나 복잡하더라도 가상 주소가 아닌 대한민국 공식 5자리 새 우편번호(`Zip_Code`)를 백엔드에서 실시간으로 정합성 있게 보정해냅니다.
- **카카오 연동 진단 도구:** UI 내에 직접 특정 임의 주소를 기입해 카카오 서버와의 연독 상태, 키 유효성, IP 접근 허용 등을 간이로 자체 테스트할 수 있는 진단 패널을 개발해 편의성을 높였습니다.

### ③ 대량 업로드 및 엑셀 내보내기 확장 (`src/App.tsx`)
- **다중 파일 일괄 드래그 앤 드롭:** 여러 개의 사업자등록증 파일을 끌어다 놓거나 선택하여 백그라운드 큐 시스템을 통해 차례대로 자동 분석합니다.
- **거래처 정보 엑셀 다운로드 (엑셀 변환):** 
  - 완료된 데이터를 하나의 통합 엑셀 마스터 파일로 간편하게 내보낼 수 있습니다.
  - 사용자의 교정 요구사항을 적용하여 기존에 다소 공학적이었던 `[성공 항목 엑셀 다운로드]` 버튼 타이틀을 실제 비즈니스 도메인 용어에 맞게 **`[거래처 정보 엑셀 다운로드]`**로 수정 완료하였습니다.
- **개별 클립보드 복사 및 재시도 기능:** 개별 실패 항목에 대한 지능형 재시도(Retry) 링크 배치 및 개별 추출 정보의 간편 텍스트 카피 기능이 제공됩니다.

### ④ 일시적 부하 및 안전성 장치 (Retry Policy)
- **지수 백오프 방식의 자체 재시도 (Exponential Backoff):** Gemini API 호출 도중 일시적 할당량 부족이나 서버 과부하 상황(HTTP 429 및 503)이 인지되는 경우, 최대 5회동안 점진적으로 대기시간을 늘려가며 요청을 완벽하게 재시도하여 인공지능 분석이 중단되는 문제를 완화했습니다.

## 3. 🎨 시각적 디자인 & 사용성 (UX/UI)
- **인터페이스 테마:** 높은 가독성을 선사하도록 디자인된 심플하면서도 신뢰도가 높은 **Indigo Accent** 기반의 고대비 화이트 테마(Light Mode)를 적용했습니다.
- **일관된 폰트 시스템:** 시스템 폰트와 매끄럽게 연결되는 고가독성 현대식 고딕 산세리프 서체를 주조로 활용하여 문서 대조 효율을 극대화했습니다. 
- **컴팩트한 배치:** 좌측에는 업로드 및 처리 프로세스를 한눈에 파악할 수 있는 진행 상황 카드를, 우측에는 정밀 추출된 상세 데이터를 대조하고 복사할 수 있는 디테일 시트를 일체형 단일 레이아웃으로 디자인해 이동 최소화 동선을 구축했습니다.

## 4. 📂 파일 내역 및 프로젝트 구조
- `server.ts`: 백엔드 Express 서버. Multer를 통한 버퍼 스트리밍, Gemini 3.5 Flash 스키마 분석 인터페이스, 익스포넨셜 백오프 설계 및 주소 정제 알고리즘, 카카오 API 통신 단축 채널 구성.
- `src/App.tsx`: React 싱글 페이지 애플리케이션. 파일 큐 핸들링, ZIP 아카이빙(JSZip), 카카오 디버그 테스트 모듈 탑재, 다이내믹 UI 및 **'거래처 정보 엑셀 다운로드'** 기능 제공.
- `metadata.json`: 앱 정보 메타데이터 및 클라우드 권한 정의서.
